Current Working Bug List

Bugs in the kernel

Edit

  • Fill region is much too slow.

EditSearch

  • Hitting the "change" or "change and find" buttons with "change all" enabled should force a search to occur. Currently, it just changes the selection, if there is one.

EditSpell

  • Spelling checker should ignore single letter words.

HTML

  • Bulleted lists are not very well aligned on the left when there is more than one line. See surfit for alternative implementation.
  • In "img" directives, align="right" causes an error.
  • A <p> after an image is ignored.
  • If you have the source code for an HTML file open, and you try to open with HTML file with the "Back" command, you get an error message that is basically wrong, since there is no problem.
  • The following is not formatted correctly: 
    <dt>Raza Ahmed
<dt><a href="http://ptolemy.eecs.berkeley.edu/~sunil">Sunil Bhave</A>
<dt><a href="http://ptolemy.eecs.berkeley.edu/~wtc">Wan-Teh Chang</a>

ListBrowser

  • Select an item, then scroll down one page and select another item. Instead of replacing the original item with the new one, the new item and a "\n" are appended to the previous item.

Query

  • -entrywidth should be respected by all types of queries. E.g., it is ignored in the following: 
        ::tycho::Query .bb -entrywidth 70
    .bb mediated color {Color:} red {::tycho::querycolor {Choose color}}
    .bb mediated font {Font:} fixed ::tycho::queryfont
    ::tycho::post [::tycho::Dialog::wait .bb]

EditTcl

  • Indenting confused by a single line with "if {" and nothing more.

EditC

  • Indenting needs alot of work.
