Tycho Copyrights

Please don't check this file in with SCCS, or the SCCS variables will not be visible.

Last Updated -[Sun Jul 2 23:12:11 1995 by cxh]-

All files should have copyrights. We use two different forms of copyrights, a shorter one for interpreted files, and a longer one for compiled files.

Please see the Tycho style guide for further details about copyrights

SCCS and copyrights

Currently we are using the %Q% (percentQpercent) sccs directive to adjust the copyright dates automatically. The copyright should have %Q% as the ending date of the copyright: 1990-%Q% If you are setting up a new directory, then you should do: 
	sccs admin -fq1995 SCCS
to set the Q variable in the SCCS directory. Note that after doing the sccs admin command, 'make sources' will check out all the files in that directory. If you are adding a new file, you can do: 
	sccs admin -fq1995 SCCS mynewfile.cc
See ~ptdesign/adm/copyrights for the script that can automatically updated copyrights.

Copyright for interpreted files

Interpreted files, such as tcl files should have the following copyright (Note that makefiles should not have this copyright, rather, they should have the copyright for compiled files, since makefiles are likely to actually be read) # One line description of what is this file. # # Author: XXX # Version: $Id$ # # Copyright (c) %Q% The Regents of the University of California. # All rights reserved. # See the file $PTOLEMY/copyright for copyright notice, # limitation of liability, and disclaimer of warranty provisions. Next year we will need to change "%Q%" to 1995-%Q%, but I guess the string is sufficiently unique that this should be OK.
