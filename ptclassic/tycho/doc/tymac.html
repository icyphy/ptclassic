    • Tycho/Macintosh bugs

    The prebuilt Itcl2.2 Macintosh binaries will not work with Tycho0.2beta. Tycho0.2 will work with an as-yet unreleased version of Itcl2.2p2 on the Mac. We do not know if or when Itcl2.2p2 on the Mac will be released.

  • Tycho 0.1.1 did not work under Itcl2.2 apparently because when Tycho started up the stack size was so large that it crashed the Mac. With Itcl2.2p2, Tycho 0.1.1 and Tycho 0.2 both seem to work.
  • Tycho0.2 does not handle spaces in pathnames very well. Tycho will work best if the distribution is installed in a directory that has a pathname that contains no spaces.
  • Since the Mac has different filename conventions, not all the filename facilities work properly. See the Tcl filename man page for information about pathnames on the Mac.
  • Cascade menus under Itcl2.2p2 and under Tk4.2 on the Mac seem to be broken. It is very difficult to use the mouse on a cascade menu, such as File -> Tools. The workaround is to use the Up, Down and Arrow keys to invoke menu choices.
  • Starting Tycho requires sourcing $TYCHO/lib/mac/tycho-mac script by hand. There must be a better way.
  • Tycho on the Mac should use an installer.

    • Installing Tycho on the Macintosh

    1. The Tycho home page at http://ptolemy.eecs.berkeley.edu/tycho/Tycho.html has a link to the downloads page that contains Tycho sources. When Itcl2.2p2 is released, it will be available on that page as well.
    2. Download both the Tycho distribution and the Itcl distribution and place them in directories next to each other. For example,
    3. The Tycho distribution could be in mac:devel:tycho0.2
    4. The Itcl distribution could be in mac:devel:Itk2.2p2-ppc
    5. Install Itcl by following the instructions in the README-mac file.
    6. Move the tycho-mac script from tycho0.1.1:lib:mac to the top level Itcl directory. In this example, tycho-mac would end up at mac:devel:itcl2.2-ppc:tycho-mac.
    7. To start up Tycho, start Itkwish, then type source tycho-mac.
    8. Tycho includes HTML documentation of the Itcl classes that make up Tycho. To save space, these files are not shipped in the Tycho tar file, you must build them after untarring the distribution.

      The BuilderDialog is an experimental graphical interface to the build process. The BuilderDialog interface will change over time, but you may find it useful for building the HTML documentation. To run the BuilderDialog, choose the Build Tycho submenu choice under the Tools menu choice in the main Window menu.

      Under Unix, the best way to build these files is to run make install. However, under Windows, a make binary may or may not be present, so running the BuilderDialog is usually the best option.
