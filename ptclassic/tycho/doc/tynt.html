    • Installing Tycho Under NT

    1. Grab the tar file from http://ptolemy.eecs.berkeley.edu/tycho
    2. Ungzip and untar the distribution using your favorite utility. I used GNU tar from the Cygnus GNU-win32 tools.
    3. Tycho refers to the directory where the Tycho distribution is as $TYCHO. In unix, under csh, you would do something like setenv TYCHO /usr/local/tycho. Under NT, you can set the TYCHO variable by clicking on Start, then Settings, then Control Panel and finally the System icon. In the System Properties window that comes up, select Environment. There, you can add TYCHO as a User Variable. Incidentally, this is where you set your path.
      To create a script to start tycho under NT, I downloaded the distribution as c:\users\cxh\tycho0.1.1. Then I created a script called c:\users\cxh\bin\ty.itcl by using the Notepad Utility. ty.itclhad the contents: set env(TYCHO) c:/users/cxh/tycho0.1.1 source $env(TYCHO)/kernel then I clicked on the ty.itcl icon, and when I was prompted to select an application, I chose c:/bin/itkwish2.1
    4. The Tycho tydoc utility converts Itcl source files to html documentation. The html code documentation is not shipped as part of the tycho tar file. To create the documentation, you have two choices:
      1. Convert each Itcl doc file by opening it up with tycho, and then selecting the gendoc option under the Special menu.
      2. If you have GNU make installed, you can convert all the Itcl files by first editing $TYCHO/lib/tydoc/tydoc and commenting out the line that contains the exec command at the top, and then running make in the tycho directory: 
        	make TYDOC="c:/bin/itclsh21 c:/users/cxh/tycho0.1.1/lib/tydoc/tydoc" itcldocs

    Tycho/NT bugs

    1. In the default NT setup, exec rm does not work, there is no rm binary. Ditto with exec mkdir. If you download the Cygnus tools. you will have mkdir and rm.
