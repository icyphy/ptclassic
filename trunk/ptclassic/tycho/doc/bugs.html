Bugs in the kernel

MenuBar

  • If you add a menu with a name that already exists, and the existing menu is before an entry that has no -label option (such as a separator), then the add will fail with an error. The reason is that the menubar uses the menu label as the only way of accessing menu entries, and separators have no label! The design is flawed, but it's probably not worth fixing this until we find out what support Tk 8.0 has for menubars, and using that.

File

  • File names are being added to the list of open files, but if the constructor subsequently fails, then Tycho becomes incoherent. It looks like the File methods used by openContext barf because a widget is in the file index but doesn't exist.

Displayer

  • The menu buttons created in MenuSupport and MenuBar may need to be made global (with an uplevel #0).

Edit

  • Fill region is much too slow.
  • The undo in the text widgets is only one level deep.
  • Meta-f, Meta-d do not skip over white space (a Tk "feature" -- need to override it).
  • Need quick font-size control in the menus, like in HTML.
  • Need a "find-again" binding, and perhaps emacs-style incremental search.
  • Selecting text does not automatically put it into the clipboard. After it's deselected, then it cannot be "stuffed" elsewhere from the clipboard.

EditSearch

  • Hitting the "change" or "change and find" buttons with "change all" enabled should force a search to occur. Currently, it just changes the selection, if there is one.

EditSpell

  • Spelling checker should ignore single letter words.

File

  • Crash recovery code only works with Ptolemy binaries.

FileBrowser

  • Name completion does not work through directories, you can only complete names of files in the current directory, not names in the sub directories.

Glimpse

  • Glimpse should not freeze Tycho while running. Searching through say the Ptolemy tree can take a very long time.
  • Glimpse.itcl should probably not be in the kernel.
  • If $TYCHO/.glimpse_exclude is not present, then we should as the user if we want to create it.

HTML

  • If I create a new Java template and generate the doc the indenting is all wrong, because the file contains HTML like: 
      <dl>
    <dt> <b>Parameters:</b>
    <dd> parameter-name - description
    <dd> parameter-name - description
    <dt> <b>Returns:</b>
    <dd> description
  </dl>
    Tycho's HTML viewer doesn't deal with the double <dd> (Netscape does, BTW).
  • The HTML parser is quite slow.
  • In "img" directives, align="right" causes an error.
  • In "img" directives dollar signs are not properly handled. The HTML code: 
    <img src="$PTOLEMY/tycho/typt/kernel/img/ptolemyWelcome.gif" >
    fails:
  • A <p> after an image is ignored.
  • If you have the source code for an HTML file open, and you try to open with HTML file with the "Back" command, you get an error message that is basically wrong, since there is no problem.
  • If I have two HTML viewers open and if in one of them I click on a link that references the file open in the other, the other one is brought to the font and the one I am looking at doesn't change. This isn't what some people would expect.
  • HTTP is not supported, so hyperlinks can only be to local files.
  • There should be a menu entry to hide the toolbar and location.
  • Not all toolbar commands are available from menus.
  • There are no key bindings for toolbar commands.
  • The print command prints only a crude facsimile of the document.
  • Need to support forms (most of the infrastructure is there already).
  • Need to support mailto, gif files, etc.
  • The following two line html file causes an error: 
    </pre></tcl>
</pre></tcl>
  • The following one line html file causes an error because the backslash precedes the </CODE>. 
    <CODE>c:\Program Files\tycho0.1.1\</CODE>
    The short term workaround is to add a space: 
    <CODE>c:\Program Files\tycho0.1.1\ </CODE>
  • If you Open Source and then do a Save As in the source file window to save the source as a different file, then executing Open Source in the HTML viewer again raises the new file instead of re-opening the source file.
  • If an HTML viewer and editor are open on the same file, then there is no window menu entry for the editor. This is a mis-feature (hjr): when editing a bunch of HTML source files, I should be able to get to the sources without having to bring the viewer to the front and then bring the editor up from behind it.
  • It would be nice if there was a simple way of opening an HTML editor on a file. Perhaps we need a way of registering _multiple_ editors for a file extension and have a popup menu in the file browser that selects the type of editor.
  • The status line usually displays where the hyperlink under the mouse goes. If you have more than one hyperlink in a menu arranged vertically, then the status line is not updated as you move the mouse vertically. See the menu at the top of tynt.html.

Query

  • Control-v in the search window inserts garbage into the search field.

RevControlRCS

  • Unfortunately, if RCS does not necessarily put the version control files in the RCS subdirectory. If the RCS subdirectory does not exist, then the directory is not created and the version control file is put in the current directory.

RevControlSCCS

  • Revision control appears to use the file permissions to test whether a file is checked out. To see, check out a file, change permissions to 444, and open the rev control window in Tycho. It should check for the lock file.

TclShell

  • When a TclShell is displaying stdout on its last line, typed commands are not always interpreted correctly.
  • Meta-< is not interpreted correctly: it puts the insertion cursor at the top of the shell. It should only put the view up there. It needs to run adjustInsertionCursor.

WelcomeWindow

  • If your ptolemy binary has a long name, then the welcome window is cropped at the bottom, and there is no scrollbar. We don't generally want a scrollbar here, and either the space bar or C-x v will scroll down (or the page down or down arrow buttons), so it's not clear this is a bug.

Bugs in text editors

EditHTML

  • Hyperlinks to other files are not composed with the directory of of the current file. Instead, it uses the pwd.
  • The Check Hyperlinks command flags every http reference as an error. Since it can't do HTTP yet, it should ignore these.

EditProgram

  • Indent region should be undoable.
  • Spelling checker should ignore HTML directives in comments.
  • When generating docs, if the documentation directory is not writable, the user should be asked if they want to create their own doc directory.

EditTcl

  • Indenting fails to ignore backslash on the end of a comment line.
  • Indenting confused by a single line with "if {" and nothing more.

EditC

  • Indenting needs a lot of work.

Bugs in visual editors

There are lots of bugs to be fixed in the visual editors, here are some of the biggies.

Graphics

  • Cut and past does not work in the Graphics widget.
  • Moved titles new position is not saved.
  • Moving or adding titles does not mark the buffer modified.
  • Most bindings in the File menu do not work.

TclProfile

  • The TclProfile probably won't work under Windows unless TclX is installed by hand. The lib/profile directory probably won't help much.
  • Sometimes 4 colons instead of 2 colons are printed.

Bugs in the documentation generator

  • The gendoc command leaves a huge number of ::tycho::DocClass objects around after completing. Do "info objects" after running gendoc to see them.

Ptolemy/Tycho bugs

  1. If you start tycho -ptiny and put it in the background, the Matlab interface might not start up on SystemV operating systems such as Solaris and HPUX. This workaround is to start tycho -ptiny >/dev/null &
