Tycho's default layout

This is the standard layout of a full Tycho installation. If you install Tycho together with a number of packages, then you will get a subset of this layout. You can also install the Tycho meta-package only, and then add individual packages to the directory as you see fit. In this case, you can place packages anywhere you like, as long as the package searcher can find them.

In the standard installation, packages are not installed in the top-level directory, but underneath sub-directories called kernel/, edit/, util/. This provides for manageable growth in the number of packages. The structure of each package is given in Package structure.

By convention, each package that is a proper part of Tycho has a name such as tycho.kernel.gui, to mimic the Java convention of placing packages by directory. The Tcl package system, of course, does not understand this convention, so in a non-standard installation you can place the packages anywhere you please. Some packages in the util directory are completely stand-alone, and so have simple names such as notmake.

Note that you do not need to install the Tycho framework to use Tycho packages. As long as you have all the packages you require, you can install them in your own directories in any way you choose.

The admin directory, containing release notes, announcements, and so on. This directory is not shipped with the rest of the distribution
dist/
The distribution directory. The contents of this directory are automatically generated.
bin/
Shell scripts that start the tclsh, itclsh, wish, or itkwish, load one or more packages, and do something useful. Many of the scripts in this package will be copied into this directory from the bin/ directories of individual packages.

basic/
the minimal set of classes that is used by any non-graphical Tycho program, and includes Tcl procedures that could be used anywhere, such as list and string manipulation. Its bin/ directory contains scripts that know how to start Tycho up in various graphical and non-graphical configurations.
gui/
the Tycho graphical kernel, which is used by all graphical applications.
html/
contains the HTML viewer and support classes such as HTTP access and network support.
model/

tycommon.mk
Contains rules to build files from source files.

util/
A meta-package containing packages of general utility. Most of these are not part of Tycho proper, but are useful stand-alone tools.
