User's Guide to the Tycho Revision Control

The RevControl class in Tycho provides a user friendly interface for revision control for RCS and SCCS.

The Edit class provides the following features:

  • Checkin/CheckoutThese are the most commonly used commands.
  • UneditThis is used to undo the file that has been checked out.
  • View HistoryThis will open up all of the history associated with the file, and provide hyperlinks to the older revisions.
  • Diff Checked FilesThe user can check the different revisions to compare using this command. A diff editor will pop up and color the differences between the files, while leaving the common text in default color. When View History is first invoked, by default, the two most current files will be checked.
  • Undo Last CheckinWhen this is invoked, the most recent check-in will be undo. In another word, the user will check out the most recent file, and delete the log from history so that he/she can continue editing. Beware that the combination of Diff Checked Files followed by an Undo implies erasing the most recent revision.
